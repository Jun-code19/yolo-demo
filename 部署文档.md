# YOLO 目标检测系统 - 部署文档

## 概述
本文档旨在提供 YOLO 目标检测系统的详细部署指南，涵盖在线部署和离线部署两种场景。

## 一、在线部署

本系统推荐使用 Docker Compose 进行部署，这可以大大简化环境配置和服务管理。

### 1. 环境要求
- Docker 20.10+
- Docker Compose 1.29+
- （可选，用于GPU加速）NVIDIA显卡驱动及 NVIDIA Container Toolkit (参考 [NVIDIA Docker安装指南](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html) 进行安装)
- 网络摄像机 (支持 RTSP 协议)

### 2. 代码获取
您可以选择通过 Git 克隆项目，或者直接拷贝项目代码到您的部署目录。
```bash
git clone <your_repo_url>
# 如果已下载，请确保您在项目根目录
cd yolo-demo
```

### 3. Docker Compose 部署

#### 3.1 准备 Docker Compose 文件
在项目根目录下确保存在以下文件，通常它们位于 `配置文件-cpu/` 目录中：
- `docker-compose.yml`
- `Dockerfile.data-server`
- `Dockerfile.detect-server`
- `Dockerfile.frontend`
如果这些文件不在项目根目录，请将其复制过来。

#### 3.2 GPU 支持配置 (可选)
如果您的服务器配备了 NVIDIA GPU 并希望利用其进行推理加速，需要进行以下修改：

1.  **安装 NVIDIA Container Toolkit**：
    请参考 [NVIDIA Docker安装指南](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html) 进行安装。

2.  **修改 `Dockerfile.detect-server`**：
    *   删除或注释掉 `ENV CUDA_VISIBLE_DEVICES=-1 FORCE_CPU=1` 一行。
    *   根据您的 CUDA 版本，将基础镜像从 `python:3.10-slim` 更改为 NVIDIA 官方 CUDA 镜像（例如 `nvidia/cuda:11.7.1-cudnn8-runtime-ubuntu22.04`）。

3.  **修改 `docker-compose.yml`**：
    在 `detect-server` 服务的配置中添加 `deploy` 或 `runtime` 配置，以允许 Docker 容器访问 GPU。
    ```yaml
    detect-server:
      # ... 其他配置 ...
      deploy:
        resources:
          reservations:
            devices:
              - driver: nvidia
                count: all
                capabilities: [gpu]
    ```
    或者使用旧版 `runtime` 配置 (仅适用于 Docker Engine 19.03 及更高版本)：
    ```yaml
    detect-server:
      # ... 其他配置 ...
      runtime: nvidia
    ```
    请根据您的 Docker 版本和需求选择合适的配置。

#### 3.3 数据库和环境变量配置
-   **数据库**: `docker-compose.yml` 中已包含 PostgreSQL 数据库服务。您可以修改 `postgres` 服务下的 `environment` 变量来配置数据库用户、密码和名称。
-   **其他环境变量**: 对于 `data-server` 和 `detect-server`，可以在 `docker-compose.yml` 文件的 `environment` 部分配置环境变量（如 `DATABASE_URL`，`LOG_LEVEL` 等）。

#### 3.4 启动服务
在项目根目录下执行以下命令启动所有服务：
```bash
docker-compose up -d --build
```
-   `up`: 创建并启动容器。
-   `-d`: 在后台运行容器。
-   `--build`: 强制重新构建镜像，确保最新的代码和配置生效。

#### 3.5 访问服务
-   **前端界面**: 通常在 `http://localhost`。
-   **检测服务 API 文档**: `http://localhost:8000/docs`
-   **数据服务 API 文档**: `http://localhost:8001/docs`

#### 3.6 停止和管理服务
-   **停止所有服务**：
    ```bash
    docker-compose down
    ```
-   **停止并删除数据卷**：
    ```bash
    docker-compose down -v
    ```
-   **查看服务状态**：
    ```bash
    docker-compose ps
    ```
-   **查看服务日志**：
    ```bash
    docker-compose logs -f <service_name>
    ```
    例如: `docker-compose logs -f detect-server`

## 二、离线部署

### 1. 离线包制作

请参考 `配置文件-cpu/create_windows_offline_package.bat` 和 `配置文件-cpu/create_linux_offline_package.bat` 脚本来制作Windows和Linux的离线安装包。

主要步骤包括：
- **构建Docker镜像**: 针对数据服务、检测服务、前端服务分别构建Docker镜像，并确保PostgreSQL镜像可用。
  ```bash
  # 示例: 构建数据服务镜像
  docker build -f Dockerfile.data-server -t yolo-data-server .
  ```
- **保存Docker镜像**: 将所有构建好的Docker镜像保存为.tar文件，以便离线加载。
  ```bash
  docker save yolo-data-server -o docker_images/data-server.tar
  ```
- **复制项目文件**: 复制项目代码、Dockerfile、配置文件 (如 `requirements.txt`, `nginx.conf`) 以及数据目录 (`models`, `storage`, `uploads`) 到离线包目录。
- **创建离线版 docker-compose.yml**: 生成一个 `docker-compose.yml` 文件，其中服务的 `image` 字段直接引用已保存的镜像，而不是通过 `build` 指令构建。
- **创建安装脚本**: 编写 `install.bat` (Windows) 或 `install.sh` (Linux) 脚本，用于加载Docker镜像、创建必要的目录并使用 `docker-compose up -d` 启动服务。
- **创建管理脚本 (Linux)**: 编写 `manage.sh` 脚本，提供 `start`, `stop`, `restart`, `status`, `logs` 等管理命令。
- **创建卸载脚本 (Windows)**: 编写 `uninstall.bat` 脚本，用于停止并删除Docker容器和镜像。
- **创建说明文档**: 在离线包中包含一个 `README.md`，详细说明离线安装的系统要求、安装步骤、访问地址和常用命令。
- **打包压缩**: 将整个离线包目录压缩成 `.tar.gz` 或其他适合传输的格式。

### 2. 离线安装

#### 环境要求
- Docker 20.10+
- Docker Compose 1.29+
- 目标机器具有足够的内存和磁盘空间。

#### 安装步骤
1.  **解压安装包**: 将制作好的离线包传输到目标机器上，并解压到指定目录。
    ```bash
    # Linux 示例
    tar -xzf yolo-linux-x64-offline_YYYYMMDD_HHMMSS.tar.gz
    cd yolo-linux-x64-offline_YYYYMMDD_HHMMSS
    ```
2.  **设置执行权限 (Linux)**: 如果是 Linux 系统，给安装脚本添加执行权限。
    ```bash
    # 检查文件格式
    file install.sh

    # 如果显示"with CRLF line terminators"，需要转换格式，安装dos2unix工具
    sudo apt-get update
    sudo apt-get install dos2unix

    # 转换文件格式
    dos2unix install.sh
    dos2unix manage.sh

    chmod +x install.sh manage.sh
    ```
3.  **运行安装脚本**: 执行安装脚本。
    ```bash
    # Windows
    install.bat
    # Linux
    ./install.sh
    ```
    脚本将自动检查Docker环境、加载Docker镜像、创建必要目录并启动所有服务。
4.  **等待服务启动**: 安装脚本会等待一段时间，确保所有服务成功启动。

### 3. 服务启动

离线安装完成后，服务应该已经自动启动。您可以通过以下命令检查服务状态：
```bash
docker-compose ps
```

如需手动管理服务，可以使用以下命令：
```bash
# 启动服务
docker-compose up -d
# 停止服务
docker-compose down
# 重启服务
docker-compose restart
# 查看实时日志
docker-compose logs -f
```
对于 Linux 系统，可以使用 `manage.sh` 脚本进行更便捷的管理。
